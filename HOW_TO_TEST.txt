WHAT IS THIS APPLICATION?
--------------------------
This is a Bus Charging Scheduler that helps schedule electric buses charging 
along a fixed route. It optimizes when and where buses should charge while 
respecting battery limits and charger availability.

Think of it like: Multiple buses traveling from Bengaluru to Kochi need to 
stop at charging stations (A, B, C, D) along the way. The app figures out 
the best charging schedule to minimize wait times.


================================================================================
HOW TO START THE APPLICATION
================================================================================

1. INSTALL REQUIREMENTS (First Time Only)
   -----------------------------------------
   Open your terminal/command prompt and run:
   
   pip install -r requirements.txt
   
   This installs all needed libraries (streamlit, pydantic, plotly, pandas)


2. START THE APPLICATION
   ----------------------
   In your terminal, run:
   
   streamlit run app.py
   
   The app will automatically open in your web browser at:
   http://localhost:8501
   
   If it doesn't open automatically, copy that URL into your browser.

HOW TO TEST - STEP BY STEP
================================================================================

TEST 1: BASIC FUNCTIONALITY
---------------------------
Goal: Verify the app loads and runs

Steps:
1. Start the app (streamlit run app.py)
2. You should see the main page with title "Bus Charging Scheduler"
3. Look at the left sidebar - you should see a dropdown menu
4. The dropdown should show 5 scenarios:
   - Scenario 1 - Even Spacing
   - Scenario 2 - Bunched Start
   - Scenario 3 - Asymmetric Load
   - Scenario 4 - Operator Heavy
   - Scenario 5 - Worst Case Convergence

Expected Result: ✓ App loads without errors
                 ✓ All 5 scenarios appear in dropdown


TEST 2: RUN A SIMPLE SCENARIO
------------------------------
Goal: Generate a charging schedule

Steps:
1. Select "Scenario 1 - Even Spacing" from dropdown
2. Click on "Scenario Configuration" tab
3. You should see:
   - Battery Capacity: 240 km
   - Charge Duration: 25 minutes
   - Route: Bengaluru → Kochi
   - List of buses with their departure times
4. Click the "▶️ Run Scheduler" button in the sidebar
5. Wait a few seconds (you'll see a spinner)
6. You should see "✅ Scheduling complete!" message

Expected Result: ✓ Scenario loads correctly
                 ✓ Scheduler runs without errors
                 ✓ Success message appears


TEST 3: VIEW BUS TIMELINES
---------------------------
Goal: Check individual bus schedules

Steps:
1. After running the scheduler (Test 2)
2. Click on "Bus Timelines" tab
3. You should see:
   - 4 metric cards at top showing:
     * Total Wait Time
     * Max Wait Time
     * Average Wait Time
     * Total Buses
   - Bar charts showing wait times
   - Operator comparison charts
   - Interactive Gantt chart timeline

4. Click on any bus card to expand it
5. You should see:
   - Bus ID, operator, direction
   - Departure and arrival times
   - List of charging stops with:
     * Station name
     * Arrival time
     * Wait time (if any)
     * Charging start and end times

Expected Result: ✓ All buses are listed
                 ✓ Each bus shows complete journey
                 ✓ Charging stops are clearly shown
                 ✓ Wait times are displayed


TEST 4: VIEW STATION QUEUES
----------------------------
Goal: Check what happened at each charging station

Steps:
1. After running the scheduler
2. Click on "Station Queues" tab
3. You should see:
   - Utilization charts at the top

     * Bus ID
     * Arrival time
     * Wait time
     * Charge start time

Expected Result: ✓ Each station shows its queue
                 ✓ Buses are listed in chronological order
                 ✓ Wait times are highlighted (yellow background)
                 ✓ Metrics are accurate


TEST 5: TEST ALL 5 SCENARIOS

Goal: Ensure the schedule is valid

Steps:
1. Run any scenario
2. Go to "Bus Timelines" tab

   - If a bus arrives while another is charging, it waits

                 ✓ Stations are visited in correct order
                 ✓ No charger conflicts (overlapping times)

================================================================================
WHAT EACH SCENARIO TESTS
================================================================================

SCENARIO 1: Even Spacing
- Buses depart evenly spaced throughout the day
- Tests basic scheduling with minimal conflicts
- Expected: Low wait times, balanced station usage

SCENARIO 2: Bunched Start
- Multiple buses depart around the same time
- Tests handling of simultaneous arrivals at stations
- Expected: Some buses will wait, queuing behavior visible

SCENARIO 3: Asymmetric Load
- Uneven distribution of buses across operators
- Tests operator fairness objective
- Expected: Scheduler balances wait times across operators

SCENARIO 4: Operator Heavy
- One operator has many more buses than others
- Tests extreme operator imbalance
- Expected: Fairness objectives prevent one operator from dominating

SCENARIO 5: Worst Case Convergence
- Designed to create maximum contention
- Tests scheduler under stress
- Expected: Higher wait times, but still valid schedule


================================================================================
COMMON ISSUES AND SOLUTIONS
================================================================================

ISSUE: "Port 8501 is already in use"
SOLUTION: Either:
  - Close the other Streamlit app running on that port
  - Or run: streamlit run app.py --server.port 8502

ISSUE: "Module not found" error
SOLUTION: Install requirements:
  pip install -r requirements.txt

ISSUE: "No scenario files found"
SOLUTION: Make sure you're running the app from the project root directory
  where the "scenarios" folder exists

ISSUE: Scheduler takes too long
SOLUTION: This is normal for complex scenarios. Wait up to 30 seconds.
  If it takes longer, there may be an issue.

ISSUE: Charts don't display
SOLUTION: Make sure you clicked "Run Scheduler" first. Charts only appear
  after a schedule is generated.

WHAT TO LOOK FOR (EXPECTED BEHAVIOR)
================================================================================

GOOD SIGNS:
✓ All buses complete their journey
✓ No bus runs out of battery (never exceeds 240 km between charges)
✓ Charging stops are in route order (A → B → C → D)
✓ Only one bus charges at a station at a time
✓ Wait times are reasonable (usually 0-30 minutes)
✓ Different scenarios produce different results
✓ Changing weights in scenario files changes the schedule

BAD SIGNS (Report These as Bugs):
✗ App crashes or shows error messages
✗ Bus travels more than 240 km without charging
✗ Stations visited out of order
✗ Two buses charging at same station at same time
✗ Negative wait times
✗ Bus never reaches destination
✗ Charts show "NaN" or undefined values

ADVANCED TESTING (Optional)
================================================================================

TEST: Modify Scenario Weights
------------------------------
1. Open scenarios/scenario1.json in a text editor
2. Find the "weights" section:
   "weights": {
     "individual": 1.0,
     "operator": 1.0,
     "overall": 1.0
   }
3. Change values (e.g., set "individual": 5.0)
4. Save the file
5. Reload the scenario in the app
6. Run scheduler
7. Compare results with original weights

Expected: Different weight values produce different schedules


TEST: Add More Chargers
------------------------
1. Open scenarios/scenario1.json
2. Find a station, e.g.:
   {"id": "B", "name": "Station B", "num_chargers": 1}
3. Change to: "num_chargers": 2
4. Save and reload
5. Run scheduler

Expected: Less waiting at Station B (more chargers available)
